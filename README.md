# p0-w4-d3-t4 · Плавное движение через requestAnimationFrame

> Тип: объёмная (rAF-анимация) · Репозиторий: один отдельный

> Подключить `.js` к HTML через `<script>`, отсмотреть в браузере.

## Задача

Двигать элемент по экрану можно на `setTimeout`, но получится рвано и не в такт с браузером. Для анимации есть `requestAnimationFrame` — браузер сам зовёт твою функцию перед каждой перерисовкой, ~60 раз в секунду. Сегодня двигаешь блок через rAF и понимаешь, чем это лучше таймера.

```
плавное движение через requestAnimationFrame
  rAF(step): браузер зовёт перед перерисовкой (~60fps)
  двигаем блок на 1px за кадр → плавно
  setTimeout для анимации — рвано, rAF — синхронно
анимируем transform, не left (дешевле для браузера)
```

## Дано

```html
<button id="run">Запустить</button>
<div id="track" style="position:relative; height:60px; background:#161b22;">
  <div id="ball" style="position:absolute; width:40px; height:40px; border-radius:50%; background:#58a6ff;"></div>
</div>
<p id="fps">—</p>
```

## Что нужно сделать (по релизам)

**Release 1 — движение rAF.** По кнопке запустить анимацию: функция `step` двигает шар слева направо, меняя `transform: translateX(...)` на несколько пикселей за кадр, и вызывает `requestAnimationFrame(step)` снова — пока не доедет до правого края. Анимировать `transform`, а не `left` (объяснить в комментарии, почему это дешевле для браузера).

**Release 2 — плавность по времени.** Сделать скорость независимой от частоты кадров: считать, сколько времени прошло с прошлого кадра (`timestamp`, который rAF передаёт в колбэк), и двигать на основе прошедшего времени, а не на фиксированные пиксели. Так анимация одинаковая на быстром и медленном экране.

**Release 3 — счётчик FPS.** Считать кадры в секунду (сколько раз rAF вызвался за секунду) и выводить в `#fps`. Показать, что при простой анимации это ~60.

## Ожидаемый результат

Шар плавно едет слева направо и останавливается у края; скорость ровная; в `#fps` — примерно 60. Анимация идёт через `transform`.

## Критерии приёмки

* Анимация построена на `requestAnimationFrame` (не `setInterval`/`setTimeout`); анимируется `transform`.
* Движение завязано на прошедшее время между кадрами, а не на фиксированный шаг за вызов.
* Счётчик FPS считает частоту кадров; ты можешь объяснить, чем rAF лучше таймера для анимаций.
